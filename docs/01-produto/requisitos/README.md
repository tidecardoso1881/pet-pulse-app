# 📋 Documentação Funcional — PetPulse

Esta pasta centraliza toda a documentação funcional do produto.  
Qualquer nova feature, fluxo ou regra de negócio deve ser documentada aqui antes de ser desenvolvida.

---

## Estrutura

| Arquivo | Descrição |
|---|---|
| `requisitos-funcionais.md` | Lista completa de requisitos funcionais por módulo |
| `fluxos-usuario.md` | Jornadas e fluxos do usuário (ex: cadastro de pet, agendamento) |
| `regras-de-negocio.md` | Regras e validações do sistema |
| `banco-de-dados.md` | Schema, tabelas e relações do Supabase |
| `design-system.md` | Paleta de cores, tipografia, componentes visuais |
| `roadmap.md` | MVP → v1.0 → próximas versões |

---

## Stack Técnico

- **Frontend:** Next.js 14 (App Router) + TypeScript
- **Estilização:** Tailwind CSS + Shadcn/ui
- **Banco de dados:** Supabase (PostgreSQL)
- **Autenticação:** Supabase Auth
- **Storage:** Supabase Storage
- **Formulários:** React Hook Form + Zod
- **Deploy:** Vercel

---

## Como usar esta pasta

1. Antes de desenvolver qualquer feature, crie ou atualize o documento correspondente
2. Use os arquivos como fonte de verdade para o desenvolvimento
3. Cole aqui o documento de requisitos gerado pelo Claude Web
4. Mantenha o `roadmap.md` atualizado conforme o produto evolui
